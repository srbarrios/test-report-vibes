# Security Policy

## Supported versions

Security fixes are applied on a best-effort basis to the latest released version.

## Reporting a vulnerability

Please do not disclose vulnerabilities in public issues.

Primary contact: `srbarrios@gmail.com`

Fallback contact: GitHub `@srbarrios`

Report security concerns privately to the maintainers by email and include:

- A clear description of the issue
- Steps to reproduce
- Impact assessment
- Suggested remediation, if available

If email is unavailable, open a GitHub issue with minimal detail and request a private channel for disclosure.

## Response expectations

Maintainers will acknowledge reports as quickly as possible and coordinate validation, remediation, and disclosure timing.
